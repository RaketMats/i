# The reflective conversation
- Always talk about the problem and how to understand it. Not understanding it, that is problem works.
- Always talk about which problem to start understanding, or solving. Then stick to it until you give up. Then talk about what to do next. Dont stop or start solving another problem, until the entire team, users, stakeholders are aware, or more preferrably agreeing. 
- Always have the conversation about the current problem ongoing. Dont have meeetings or workshops without an agenda to kick of that part of the conversation, and minutes to sum it all up. Do the same for other conversation channels such as chat groups or mail-threads, but less formal naturally.
- Always have a conversation about what and how to do. Dont ever just start doing something without first having discussed what to do. E.g. if writing a manual, dont just start writing. First talk about what it is that shall be written, for whom, which form. Dont solve, just discuss, then do, build, then reflect.
- Finally, start working on the solution. 
## N.B.
Note that the above is in line with the solution gravity well, i.e. if possible, start building something.   
Discussing it as described above, just change 'problem' with 'solution'.

# About
Project management is normally done by identifying tasks, organising them in some kind of order, estimate time. But design work can not be done that way because it is almost always 'complex' problems which need both analyse and experience and that requires a reflective, discovery way of working. Read the book 'Conceptual Blockbusting'. Study the 'Cynefin Framework'. Read the book 'Design Way' which suggest a way of working based upon 'judgement' and from that, defining 'directions'. 

# A flipped way of working
There must be a way to identify and keep track on which problem we work on. In traditional project management it is a time plan with tasks, in agile it is a backlog. Backlog is better. But an even better way is to instead provide a tool to identify where we are, and then skip planning as done in agile/project management. Instead, problems are identified, defined and this is a continous process, there is no real 'done' until the user has start using the product and is satisfied. Until then we need something else, a tool that can identify, in the entire problem space:
- the problem we currently work on
- another tool helping to evalutate how far we have come, is it defined, do we have a hypothesis, do we run experiments and learn. 
  - Note that we can pause at any moment. Everything is in the Problem and Solution space which is almost always hard to try to bring oerder, to arrange etc. as traditional project management tries to do. Instead we use these different tools to position us and help take out a new direction.
- Another tool to judge. How far have we come. Are we done/enough. 
- Another tool to evaluate result. Is the problem defined enough. Is the hypothesis clear enough to start work. How do we test/experiment. Are we running and ready enough with the experiment. How do we learn
  - All these are examples of tools that we pick upp continously, together with the continous work in Problem and Soluton space, i.e. the Design Space, and helps us build the Design Situation.
- So, it is more abouot navigation. Tools readily available and a pace/pulse of continously pulling out these tools and take our position, take our soundings, then other tools to help us take out the new direction. 
- The Design situation is all the people including users, stakeholders + Design Space which is Probleml and Solution spaces + Navigational aids.
  - the tools can be applications, or guides and manuals, or more visual infograhpics or board game mechanics or aphorisms.

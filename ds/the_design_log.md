# About
Work is done from Problem Space which describes the different needs, problems and things that just need to be done. The Problem Space is a simple textual description and which can have different formats, see examples below.  
- It is very important to define the problems
- Work shall not be started until a problem exist and is reasonable defined.
- The counterpart is Solution Space which is the learning loop, described below.  

A visual representation, 'Spaces' is also done. This is an overview of what shall be done. It can be used to get an overall presentation. It can be used to navigate the Problem Space, help understanding what need to be done.  
One of the most important purposes of the Problem Space is to provide a place to discuss what need to be done:
- Often design work 'are just done', work is just started without discussing what shall be done and how it shall be done.

- The log part is used to in a simple way, have a place to write down, or add models or sketches, about what have been done. Instead of taking time to sort, analyse and write conclusions, just write down and instead continously return to. The important things will rise to attention and can continously be rewritten. This means that work is not spent on unimportant things.

# Nota Bene
It is of utmost importance that:
- Do not mix work with breaking down the problem, in the 'Problem Space'. Keep text, descriptions, discussions etc. brief there. Instead work around what, how to do etc. the meta-discussion should be kept in other places, documents, as text, sketches, models etc. The 'Problem space' is for overview, a place to have all things that need to be done. The details shall be kept some other place. Avoid clutter.

# The Importance of a Design Situation
There must be an overall view and collection of the design work. It is important that there is a clear distinction towards project management work. The book 'Design Process och Metod'(only in Swedish) describes this well. In short, there need to be I A clear goal. II All things needed, people, tools, a place to be, a decent coffee machine. III All things that need to become, we need to have a clear picture of all things included, not only the product, but how to deliver, packaging and manuals. IV We need to own how things are built so that we have control over the entire chain. V Building the Product, we should always focus on building something, if we can not, then we shall think about it, otherwise, build.  

To help with this we can start listing some tools:
- The book 'Design Things' states the importance of 'creative density' i.e. both physical and virtual places to collect everything needed, research, reference materials, mockups and prototypes and room for people to meet.
- A 'People Model'. It is all about people, and having an overview of different roles is fundamental. A simple model is 'Users', 'the Team', and 'Stakeholders, Owner and Sponsor and Domain Experts'.
